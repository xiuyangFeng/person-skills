# 模板

字段以当前仓库 `PROJECT_PROFILE.md` 的 `required_fields` 为准。下列为通用默认。

## 推进记录（文首插入）

```markdown
## YYYY-MM-DD｜主题 · 动作 · 状态

**本次主要修改**：…

**对应代码/文档**：…

**推进到实验步骤**：…

**当前状态判断**：…

<!-- 科研可选：profile 勾选则写入 -->
**背景假设**：…

**方法变更**：…

**关键指标**：…（写明 split / seed / 对照实验；禁止跨路线裸比）

**Go-NoGo**：Go / No-Go / 待定 — 判据来源：…

**下一步**：…（尽量引用待办/主规划中的已有项）
```

### 书写规则

- 插入位置：文件头说明块之后、第一条历史记录之前。
- 一条记录一件事；大改可拆多条，但同一天同主题可合并。
- 指标必须带来源路径（run 目录、`summary.json`、图目录等）。
- 不要写「今天/昨天/最近」；状态用绝对标签。

## 正式入账（Markdown 最小集）

用于路线状态表或实验跟踪文档：

```markdown
### <exp_id>｜<一句话目的>

| 项 | 值 |
| --- | --- |
| 配置 | `path/to/config.json` |
| 数据/split | … |
| seed | … |
| 状态 | 草稿 / 已提交 / 训练中 / 完训 / 作废 |
| 关键指标 | … |
| 产物 | `outputs/.../` |
| 对照 | <baseline exp_id> |
| 结论 | … |
```

## xlsx / CSV 对齐

1. 打开 profile.`ledger.fill_spec`（规范文档）。
2. 只填规范列出的列；不知道的单元格留空并在推进记录注明「待补」。
3. 有 `csv_index` 时：保证 `exp_id`、config、输出目录三列与磁盘一致。
4. 无工作簿时：`ledger.mode=markdown`，不伪造 xlsx。

## 分析回填（短）

分析 Skill 产出结论后，本 Skill 只追加：

- 推进记录一条（含关键指标 + Go-NoGo）
- 路线跟踪日志顶块状态行（若 profile 指定了 `status_doc`）
- 待办中对应行状态（若存在且用户未禁止改待办）

禁止在此展开长篇「原因分析/曲线解读」；需要时指向分析产物目录。

## 起步骨架（scaffold）

供向导「模式 B · 从零 scaffold」在空仓库建最小可用结构。路径按用户 `docs_root` 调整；只建**最小**结构，不预设复杂分册。

### 入口 `docs/README.md`

```markdown
# 项目文档索引

> 更新时间：YYYY-MM-DD

一句话说明本项目在做什么。

## 现在先看什么（当前主线）

- **当前主线**：<一句话——正在推进的路线 + 最新结论 + 下一步>
- [推进记录](02-推进与变更/代码修改与实验推进记录.md)（只读文首 3–5 条）
- [知识库整理规范](00-规范与记录/项目知识库整理规范.md)

## 分层导航

| 层 | 位置 | 放什么 |
| --- | --- | --- |
| 入口 | 本文件 | 只导航，指向当前主线 |
| 规范 | `00-规范与记录/` | 口径、填写规范、整理规范 |
| 推进记录 | `02-推进与变更/` | 时间倒序凭证（最新在上） |
| 证据 | `outputs/` 等 | run 产物，文档只引用路径、不搬运 |
```

### 推进记录空文件 `docs/02-推进与变更/代码修改与实验推进记录.md`

```markdown
# 代码修改与实验推进记录

> 每完成一个任务/实验后，在**文首**追加一条（最新在上）。标题：`## YYYY-MM-DD｜主题 · 动作 · 状态`。
> 状态用绝对词（已提交/训练中/完训/No-Go/待验证），禁止「今天/最近」。指标必带来源路径，且不跨路线裸比。

<!-- 新记录插入到这一行下方 -->
```

### 整理规范 `docs/00-规范与记录/项目知识库整理规范.md`

```markdown
# 项目知识库整理规范

## 分层
- **入口**（README）：只导航，指向当前主线。
- **规范**（本目录）：口径、填写规范、目录约定。
- **路线状态**：每条路线一个状态主文档，唯一维护「当前状态 + Go/No-Go」。
- **推进记录**：时间倒序凭证，最新在上。
- **证据**：run 产物目录，文档只引用路径。

## 铁律
1. 同一实验状态只在一个主文档维护，别处链接或一行摘要。
2. 不跨路线裸比指标；对照必带 baseline/split/seed。
3. 结论写清 Go/No-Go 与判据来源。
4. 归档不丢证据路径（见归档清单）。
5. 不写相对时间词。
```
