# 归档 Checklist

**前置条件**：用户在当前对话明确说「归档」或等价指令。未明确则只建议，不搬文件。

## 执行前

- [ ] 确认归档范围（哪些 md / 哪条路线 / 是否含图床资产）
- [ ] 确认归档根路径（profile.`archive.roots`）
- [ ] 确认**不**移动 `outputs/`、`data*`、模型权重等证据目录（只在文档中保留路径引用）

## 执行中

- [ ] 在**当前仍活跃**的入口/主文档写清：最终结论、Go-NoGo、替代路线、证据路径
- [ ] 将过时执行清单/长篇过程文档移入 `_archive/`（或 profile 指定目录）
- [ ] 在 archive 目录放/更新 `README.md` 索引（原名、结案日期、为何归档、现行文档链接）
- [ ] 修复相对链接（归档后 `../` 层数变化）
- [ ] 推进记录文首记一条：归档了什么、结论留在哪里

## 执行后

- [ ] 现行入口不再把 archived 文档当作「当前待办」
- [ ] 搜索是否仍有正文把 archived 路径当主入口（改为链接到结论摘要）
- [ ] 向用户汇报：移动的文件列表 + 结论所在文件

## 禁止

- 因「看起来过时」自动归档
- 删除历史证据或「清理」outputs
- 归档后丢弃 Go/No-Go 与关键指标（必须留在现行文档或 archive README）
